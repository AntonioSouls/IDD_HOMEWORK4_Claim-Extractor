The user wants to extract claims from a table represented in a JSON format.
Each claim should have a "specifications", a "Measure" and an "Outcome".
The "specifications" are key-value pairs that identify the row from which the claim is extracted.
The "Measure" is a column header, potentially found in the caption or references.
The "Outcome" is the value associated with the "Measure" in that row.

Plan:
1. Identify the different measures available in the table headers.
2. Iterate through each row of the table (excluding the header).
3. For each row, extract the "specifications" based on the first column(s).
4. For each measure, extract the corresponding "Outcome" from the row.
5. Structure the extracted information into the desired JSON format.
